# Contributing

Thanks for considering a contribution.

## Quick start

1. Fork the repo and create a branch off `main`.
2. Make your change. Keep PRs focused — one logical concern per PR.
3. Run the project's tests / linters / build before pushing (see `README.md` for the canonical commands).
4. Open a PR with a brief description of the *why*, not just the *what*.

## Reporting bugs / proposing features

Open a GitHub issue. Include:
- What you expected to happen.
- What actually happened.
- Minimal repro steps if you can produce them.

## Coding standards

- Follow the existing style — match indentation, naming, and import order with neighbouring files.
- Add or update tests when you change behaviour.
- Document non-obvious trade-offs in the code or PR description.

## Code of conduct

Be respectful. Disagreement is fine; personal attacks are not.
