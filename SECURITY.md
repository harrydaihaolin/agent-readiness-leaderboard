# Security policy

## Reporting a vulnerability

If you believe you've found a security vulnerability in this project, please **do not open a public issue**. Instead:

1. Email the maintainer at <security contact in the repo's GitHub profile> with details.
2. Or use GitHub's private vulnerability reporting on this repo (Security → Advisories → "Report a vulnerability") if it's enabled.

We aim to:

- Acknowledge receipt within **72 hours**.
- Provide an initial assessment within **7 days**.
- Coordinate a disclosure timeline with you before publishing details.

## Supported versions

This project's `main` branch receives security fixes. Older releases are best-effort only.

## Out of scope

- Vulnerabilities that require physical access to a developer's machine.
- Issues in transitive dependencies — please report those upstream first; we'll mirror the fix here once it's available.
