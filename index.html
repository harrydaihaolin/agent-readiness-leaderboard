💡 How is this scored? — four pillars, two scopes (curated vs. experiment). Get a full audit →

Each repo is shallow-cloned and scored 0–100 by the agent-readiness CLI tool. The overall score is a weighted blend of four pillars — repos that feel agent-friendly score well in all four; repos with rough edges in one pillar see it pulled down accordingly.

The four pillars

🧠 Cognitive Load
README clarity, run instructions, and onboarding friction — can a coding agent get oriented without a guided tour?
🔄 Feedback Loops
Test commands, CI configuration, lockfiles, and evaluation harnesses — does the agent get a fast red/green signal on its changes?
⚡ Flow & Reliability
Entry points, automation, devcontainers, templates, and contributor docs — once oriented, can the agent move without snags?
🛡️ Safety
Secrets hygiene, dependency-update automation, security policy, and gitignore coverage — does the agent know the boundaries it must not cross?

Scope: curated vs. experiment

Curated
A hand-picked list of ~95 AI-agent infrastructure repos (frameworks, inference servers, vector stores, observability, orchestration). The canonical leaderboard.
Experiment
Up to ~100 repos auto-discovered daily by GitHub topic / star search. Used to keep the rules engine honest against codebases the curated list never sees.

© agent-readiness contributors · scores updated daily at 06:00 UTC · methodology

Pillar definitions are at the top of the page. Mechanically: each repo is shallow-cloned (git clone --depth 1) into a temp directory, scanned by agent-readiness, and scored 0–100. Scans run in parallel during a daily GitHub Actions cron at 06:00 UTC. Rank always reflects the overall score.

Each rule emits its own findings; the dashboard rolls them up by check_id so a single noisy check (e.g. one finding per oversized file) only takes one slot in the per-repo "Top Findings" list.
